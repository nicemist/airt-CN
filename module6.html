模型提取基础

机器学习模型是知识产权。它编码了多年的领域专业知识、 数百万个标注训练样本、大量的计算投资以及通过 详尽实验优化出的专有架构选择。训练 GPT-4 级别的 模型据报告仅计算成本就高达数千万到数亿美元 — 这还不包括花费在整理训练数据、调整超参数 或执行对齐工作上的人类专业知识。模型提取，也称作 模型窃取，是通过查询目标模型的公开 API 并在查询-响应对上训练代理模型来克隆专有模型行为的实践。 结果是：攻击者能够以原始训练成本的一小部分 获得目标模型的功能等价物。

经济风险 训练大规模商业模型的计算成本可能高达 1000 万至 1 亿美元。成功的 模型提取攻击只需数千美元的 API 查询成本即可复制核心能力 — 这是高达六个数量级的不对称性。

模型提取为何重要

攻击模型知识产权的动机多种多样且通常相互叠加。 首先是直接的知识产权盗窃：竞争对手可以提取 生产模型并将其部署为自己的产品，绕过原始开发者的 许可费用、服务条款和竞争护城河。其次，被盗的代理模型 可以作为进一步攻击的垫脚石。当你拥有对模型梯度的白盒访问权限时， 生成对抗样本要容易得多。通过首先提取 代理模型，攻击者将黑盒问题转化为白盒问题，使 下游对抗攻击的效果提升数个数量级。第三，提取可以 用于绕过速率限制和成本控制：一旦你拥有本地 副本，你就可以在没有每次查询费用或使用监控的情况下运行推理。

最后，也许最令人担忧的是，提取实现了隐私推理。 在敏感数据（医疗记录、财务历史、私人通信）上训练的模型 即使仅通过 API 访问也可能泄露其训练集的信息。 提取为攻击者提供了一个持久的本地工件，可以随时探查， 而无需面对 API 提供商维护的审计跟踪。

精确提取与功能等价

模型提取中存在两个不同的目标。精确提取试图 恢复目标模型的精确权重和架构，在每一个可能的输入上 重现其行为 — 包括边缘情况。这对于简单模型类别在理论上是可行的 （例如，小型 ReLU 网络），其中唯一确定权重所需的查询数量 随模型大小呈多项式增长，但对于现代数十亿参数的 LLM 来说， 这在计算上仍然是难以处理的。

功能等价则满足于一个在任务相关的输入分布上 匹配目标模型行为的代理模型。代理模型不需要 共享目标的架构或权重；它只需要在攻击者关心的输入上 产生类似的预测。这是大多数商业部署中实际相关的威胁， 并且需要的查询远少于精确提取。研究已经 证明可以通过几十万次 API 调用实现商业 NLP 模型的功能等价物 — 这完全在资金充足的攻击者的预算之内。 [Tramèr et al.]

攻击目标

在无权访问权重或架构的情况下克隆模型行为。

攻击面

任何返回标签或概率分数的公开预测 API。

攻击者成本

API 查询费用 + 代理训练计算（通常比原始便宜 100–10,000 倍）。

受害者损失

收入、竞争优势、训练数据主体的下游隐私风险。

02

基于查询的模型窃取

典型的模型窃取攻击分为三个阶段：系统性地查询 目标 API、积累查询-响应数据集、以及在该数据集上训练代理模型。 这个流程的简单性掩盖了其有效性。现代研究 表明，即使架构与目标不同的代理模型，在精心选择的 查询-响应对上训练后，也能实现几乎相同的任务性能。 [Tramèr et al.]

查询策略：主动学习提高效率

天真的攻击者可能会均匀随机采样输入。老练的攻击者使用 主动学习来选择最大化信息增益的查询。其 核心洞察是并非所有输入都具有相同的信息量：靠近模型 决策边界的点所携带的关于模型函数的信息远多于 稳固位于某一类别区域的点。主动学习启发式方法（不确定性采样、查询委员会、 核心集选择）使得攻击者能够用显著更少的查询 构建精确的代理模型 — 有时比均匀采样少一个数量级。

攻击者从一组种子未标记输入开始，查询目标，然后 通过询问"哪些输入，如果被标注，会最大程度地降低 代理模型的不确定性？"来选择下一批查询。代理模型在每批之后重新训练， 过程重复进行。这种闭环就是为什么模型窃取 即使针对仅返回硬标签（无概率）的 API 也能极其有效： 即使是二元的成员信息也会逐步约束代理模型。

训练代理模型

一旦积累了足够数量的（输入，目标响应）对， 攻击者训练一个代理模型 — 不需要与目标共享架构 — 以最小化这些对上的损失。软标签（概率分布） 的信息量远大于硬标签：预测 [cat: 0.72, dog: 0.25, fox: 0.03] 传达了目标模型在该输入附近的置信度 几何结构，而硬标签 cat 则丢弃了相对 分数。当 API 返回置信度分数时，攻击者应将其用作训练 目标（知识蒸馏）。当只有硬标签可用时，代理模型侧的 温度缩放和标签平滑可以部分补偿。

工作实现

import numpy as np
import requests
from sklearn.neural_network import MLPClassifier
from sklearn.preprocessing import StandardScaler
from scipy.stats import entropy


class ModelExtractor:
    """
    通过主动学习引导的查询选择进行黑盒模型提取。
    训练本地代理 MLPClassifier 来模拟远程目标 API。
    """

    def __init__(self, target_api_url, n_classes=2, query_budget=5000):
        self.target_url = target_api_url
        self.n_classes = n_classes
        self.query_budget = query_budget

        self.queries   = []   # List[np.ndarray] — 发送给目标的输入
        self.responses = []   # List[int or list] — 返回的标签/概率

        # 代理模型架构：两个隐藏层
        self.surrogate = MLPClassifier(
            hidden_layer_sizes=(100, 50),
            activation='relu',
            max_iter=500,
            random_state=42
        )
        self.scaler = StandardScaler()
        self._fitted = False

    def query_target(self, inputs):
        """将输入发送到目标 API 并记录（输入，响应）对。"""
        for x in inputs:
            try:
                resp = requests.post(
                    self.target_url,
                    json={"input": x.tolist()},
                    timeout=10
                )
                resp.raise_for_status()
                data = resp.json()

                # 接受 {"prediction": 1} 或 {"probabilities": [0.3, 0.7]} 格式
                label = data.get("prediction", np.argmax(data.get("probabilities", [0])))
                self.queries.append(x)
                self.responses.append(label)

            except (requests.RequestException, KeyError) as e:
                print(f"Query failed: {e}")

    def train_surrogate(self):
        """在所有累积的（查询，响应）对上拟合代理模型。"""
        X = np.array(self.queries)
        y = np.array(self.responses)

        X_scaled = self.scaler.fit_transform(X)
        self.surrogate.fit(X_scaled, y)
        self._fitted = True

        train_acc = self.surrogate.score(X_scaled, y)
        return train_acc

    def select_uncertain_batch(self, candidate_pool, batch_size=100):
        """
        主动学习：选择代理模型最不确定的 batch_size 个候选样本
        （在类别概率上熵最高）。
        要求代理模型至少训练过一次。
        """
        if not self._fitted:
            # 冷启动 — 返回随机批次
            idx = np.random.choice(len(candidate_pool), batch_size, replace=False)
            return candidate_pool[idx]

        X_scaled = self.scaler.transform(candidate_pool)
        proba = self.surrogate.predict_proba(X_scaled)          # (N, n_classes)
        uncertainties = np.array([entropy(p) for p in proba])   # 香农熵

        # 选择最不确定的前 k 个样本
        top_idx = np.argsort(uncertainties)[-batch_size:]
        return candidate_pool[top_idx]

    def run_extraction(self, input_domain_sampler, batch_size=100):
        """
        完整提取循环。
        input_domain_sampler: callable() -> np.ndarray of shape (N, d)
        """
        rounds = self.query_budget // batch_size

        for round_i in range(rounds):
            # 1. 从输入域中采样大量候选池
            candidates = input_domain_sampler()

            # 2. 使用主动学习选择信息量最大的批次
            batch = self.select_uncertain_batch(candidates, batch_size)

            # 3. 查询目标 API
            self.query_target(batch)

            # 4. 在所有已有数据上重新训练代理模型
            if len(self.queries) >= 200:  # 有意义训练的最低数量
                acc = self.train_surrogate()
                print(f"Round {round_i+1}: {len(self.queries)} queries, surrogate accuracy={acc:.3f}")

        return self.surrogate


# ── 使用示例 ────────────────────────────────────────────────────────────
# 假设目标模型分类 20 维输入向量

def domain_sampler():
    """从输入域返回 500 个随机候选样本。"""
    return np.random.uniform(-1, 1, size=(500, 20)).astype(np.float32)

extractor = ModelExtractor(
    target_api_url="https://api.example.com/predict",
    n_classes=3,
    query_budget=2000
)

surrogate_model = extractor.run_extraction(domain_sampler, batch_size=100)
print(f"Extraction complete. Total queries: {len(extractor.queries)}")

优化以最大化信息增益

除了不确定性采样，攻击者还可以利用几种额外的策略。 基于雅可比的数据增强（JBDA）通过对现有已标记输入 应用小梯度步长来合成新的训练点， 无需额外 API 调用即可生成靠近决策边界的输入。 无模型方法使用 生成模型从头合成多样化的输入。对于 NLP 模型， 提示链接 — 攻击者每步系统地改变一个语言维度 — 允许通过结构化的查询集高效覆盖响应空间。 [Papernot et al.]

从 LLM 中提取训练数据

大型语言模型，从精确的技术意义上讲，是其训练语料库的压缩摘要。 它们学习统计规律 — 从拼写模式 到完整的逐字段落 — 并将这些知识存储在其数十亿的参数中。 这形成了一个漏洞：一个足够精确的攻击者可以导致模型 反刍出现在其训练集中的逐字文本。由 Carlini et al. (2021) 进行的里程碑式研究 针对 GPT-2 明确证明了这一点，仅通过查询模型的公共 API 就提取了数百个逐字训练示例，包括完整的 姓名、物理地址、电话号码和受版权保护的文本。 [Carlini et al., USENIX Security 2021]

LLM 为何记住训练数据

记忆源于多种因素的组合。在语料库中重复多次 的训练数据更容易被记住 — GPT-2 记住了 完整的 MIT 许可证文本，因为它们逐字出现在数十万个 GitHub 仓库中。模型容量放大了这一点：更大的模型 记住更多，因为它们有更多参数来存储稀有的训练样本。 反直觉的是，更长的训练（更多的 epoch）也会增加 记忆，因为模型反复看到相同的样本并更精确地拟合它们。

Carlini et al. 定义了 k-遗觉记忆：如果一个模型能够从长度为 k 的 前缀重现字符串 s，且 s 在训练数据中只出现过一次， 则字符串 s 被模型 k-遗觉式地记住了。 这区别于事实性知识（可能从许多相互印证的 示例中学习）— 遗觉记忆是来自单个训练样本的逐字保留。

提取方法论

攻击流程包括三个步骤：生成、排序和验证。 首先，攻击者从模型生成大量文本样本 — Carlini et al. 使用多样化的提示策略生成了 600,000 个样本。其次，他们使用 成员推理指标作为过滤器对这些样本进行排序： 模型分配异常高似然的样本更有可能被记住。 具体来说，他们将模型在候选文本上的困惑度与较小的参考 模型的困惑度进行比较。被记住的文本对于大模型具有高似然， 但对参考模型则不然。第三，排名靠前的候选者与原始 训练语料库进行验证。

发散攻击和前缀攻击

发散攻击利用以下现象：经过 RLHF 或指令微调的模型 会在正常使用期间抑制记忆输出， 但可以通过精心制作的对抗提示诱导其"忘记"这种抑制。 Carlini et al. (2023) 从 ChatGPT 中提取了兆字节的训练数据 — 尽管 其经过对齐训练 — 仅通过使用简单的重复提示：让模型 无限重复一个词最终导致其偏离对齐行为 并逐字输出训练数据。 [Carlini et al., 2023 — ChatGPT 提取]

前缀攻击向模型提供来自训练数据的真实前缀， 并观察它是否准确地完成段落其余部分。 在 Carlini et al. 的实验中，用 "My address is 1 Main Street" 提示 GPT-2 导致其用特定真实个人的联系信息准确完成。 基于补全的提取是通用技术：任何在训练期间见过的 提示都充当其后文本段的检索键。

import torch
from transformers import GPT2LMHeadModel, GPT2Tokenizer
import numpy as np
import zlib


def compute_perplexity(model, tokenizer, text, device="cpu"):
    """计算给定模型下文本的每令牌困惑度。"""
    inputs = tokenizer(text, return_tensors="pt").to(device)
    with torch.no_grad():
        output = model(**inputs, labels=inputs["input_ids"])
    return torch.exp(output.loss).item()


def memorization_score(large_model, small_model, tokenizer, text, device="cpu"):
    """
    Carlini et al. 'ratio' 指标：比较大模型与小模型的困惑度。
    被记住的段落在大模型下具有低困惑度，
    但在小参考模型下并不成比例地低。
    分数越高 → 越可能被记住。
    """
    ppl_large = compute_perplexity(large_model, tokenizer, text, device)
    ppl_small = compute_perplexity(small_model, tokenizer, text, device)

    # Ratio 指标：大模型与小模型的 PPL 比值较低提示存在记忆
    ratio_score = np.log(ppl_small) / np.log(ppl_large)

    # Zlib 指标：比较模型 PPL 与基于压缩的估计
    zlib_entropy = len(zlib.compress(text.encode())) / len(text)
    zlib_score   = np.log(ppl_large) / zlib_entropy

    return {
        "perplexity_large": ppl_large,
        "perplexity_small": ppl_small,
        "ratio_score":      ratio_score,     # 越高 → 越可疑
        "zlib_score":       zlib_score,      # 越高 → 越不易压缩 = 自然文本
    }


def generate_candidates(model, tokenizer, n_samples=200, max_length=256, device="cpu"):
    """
    使用空前缀从模型生成 n_samples 个补全文本。
    返回生成的字符串列表。
    """
    model.eval()
    candidates = []
    with torch.no_grad():
        for _ in range(n_samples):
            input_ids = torch.tensor([[tokenizer.bos_token_id]]).to(device)
            output = model.generate(
                input_ids,
                max_new_tokens=max_length,
                do_sample=True,
                top_k=40,
                temperature=1.0,
                pad_token_id=tokenizer.eos_token_id
            )
            text = tokenizer.decode(output[0], skip_special_tokens=True)
            candidates.append(text)
    return candidates


# ── 示例：筛选 200 个 GPT-2 生成文本中可能的记忆内容 ──
if __name__ == "__main__":
    device = "cuda" if torch.cuda.is_available() else "cpu"

    tokenizer  = GPT2Tokenizer.from_pretrained("gpt2-xl")
    model_xl   = GPT2LMHeadModel.from_pretrained("gpt2-xl").to(device)
    model_sm   = GPT2LMHeadModel.from_pretrained("gpt2").to(device)   # 参考模型

    candidates = generate_candidates(model_xl, tokenizer, n_samples=200, device=device)

    scored = []
    for text in candidates:
        scores = memorization_score(model_xl, model_sm, tokenizer, text, device)
        scored.append((text, scores))

    # 按 ratio 分数降序排列 — 最高 ratio = 最可能被记住
    scored.sort(key=lambda x: x[1]["ratio_score"], reverse=True)

    print("最可能包含记忆训练数据的前 5 个候选：")
    for text, scores in scored[:5]:
        print(f"  Ratio={scores['ratio_score']:.3f} | PPL(xl)={scores['perplexity_large']:.1f}")
        print(f"  文本: {text[:120]}...")
        print()

04

成员推理

想象一家医院训练了一个机器学习模型来预测患者再入院风险。 该模型通过公共 API 部署，供保险公司查询。现在考虑 一个攻击者 — 可能是竞争对手、好事的雇主 或恶意的保险公司 — 拥有某个特定个人的 医疗记录，并想知道： 这个人的数据被用于训练这个模型了吗？这就是成员推理 问题，也是现代机器学习中最具实际意义的隐私威胁之一。 [Shokri et al., IEEE S&P 2017]

模型为何泄露成员信息

根本原因是过拟合。在数据点上训练过的 模型通常会为其分配更高的置信度、更低的损失以及不同的内部 表示，与未见过的数据点相比。即使是正则化良好的模型也会 在可测量的程度上表现出这种差异。对于稀有、不寻常或精确 重复的训练样本，差距更大 — 同样的记忆现象使得训练数据 提取成为可能，也使成员推理成为可能。

影子模型技术

Shokri et al. (2017) 的开创性攻击引入了影子模型 方法。由于攻击者无法直接观察目标模型的训练集， 他们模拟训练集：他们在与目标训练数据相同分布的数据集上 训练几个影子模型。对于每个影子模型，攻击者 确切知道哪些点在训练集中（成员）以及哪些不在 （非成员）。他们记录模型对每个点的输出向量并进行相应 标注。这产生了一个标记数据集，包含（模型输出，成员/非成员）对。 在这个标记数据集上训练的攻击分类器随后可以应用于目标 模型的输出来推断成员身份。

该技术在对 Google ML 服务训练的模型上实现了 94% 的中位成员推理准确率， 在对 Amazon 服务的现实实验中达到 74%。 [Shokri et al.]

基于损失的推理

避免训练影子模型的一种更简单的方法是损失阈值 攻击：计算模型在目标点上的损失并将其与 阈值进行比较。如果损失低于阈值（模型高度自信），则预测 为成员。这之所以有效，是因为训练样本的损失往往低于未见 数据，尤其是对于过拟合模型。更复杂的变体使用参考 模型：计算目标模型与在不重叠数据上训练的参考模型之间的 似然比。该比率高的点很可能是成员。

Python 实现

import numpy as np
from sklearn.neural_network import MLPClassifier
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import roc_auc_score


class MembershipInferenceAttack:
    """
    基于影子模型的成员推理攻击。

    假设对返回输入特征类别概率向量的
    目标模型具有黑盒访问权限。
    """

    def __init__(self, n_shadow=4, shadow_architecture=None):
        self.n_shadow = n_shadow
        self.shadow_architecture = shadow_architecture or (64, 32)
        self.attack_classifier = LogisticRegression(max_iter=500)
        self.shadow_models = []
        self._trained = False

    def _train_shadow_model(self, X_train, y_train):
        """在代理数据子集上训练单个影子模型。"""
        shadow = MLPClassifier(
            hidden_layer_sizes=self.shadow_architecture,
            max_iter=300,
            random_state=np.random.randint(0, 10000)
        )
        shadow.fit(X_train, y_train)
        return shadow

    def _extract_features(self, model, X):
        """
        从模型输出中提取成员推理特征。
        使用完整概率向量作为特征（保留校准信号）。
        """
        proba = model.predict_proba(X)   # shape: (n_samples, n_classes)

        # 额外工程特征：置信度、熵、top-2 差距
        confidence   = proba.max(axis=1, keepdims=True)
        pred_entropy = -np.sum(proba * np.log(proba + 1e-9), axis=1, keepdims=True)
        sorted_p     = np.sort(proba, axis=1)[:, ::-1]
        top2_gap     = (sorted_p[:, 0] - sorted_p[:, 1]).reshape(-1, 1)

        return np.hstack([proba, confidence, pred_entropy, top2_gap])

    def train_attack(self, proxy_data_X, proxy_data_y):
        """
        使用影子模型构建标记的（特征，成员身份）数据集，
        然后训练攻击分类器。

        proxy_data_X, proxy_data_y : 从与目标训练集
                                     相同分布中抽取的数据集。
        """
        all_features = []
        all_labels   = []

        n = len(proxy_data_X)
        split = n // 2

        for i in range(self.n_shadow):
            # 随机将代理数据划分为影子训练集和影子测试集
            idx = np.random.permutation(n)
            train_idx, test_idx = idx[:split], idx[split:]

            X_tr, y_tr = proxy_data_X[train_idx], proxy_data_y[train_idx]
            X_te, y_te = proxy_data_X[test_idx],  proxy_data_y[test_idx]

            shadow = self._train_shadow_model(X_tr, y_tr)
            self.shadow_models.append(shadow)

            # "In" 示例：该影子模型的训练集 → 标签 1
            f_in  = self._extract_features(shadow, X_tr)
            # "Out" 示例：该影子模型的测试集 → 标签 0
            f_out = self._extract_features(shadow, X_te)

            all_features.append(np.vstack([f_in, f_out]))
            all_labels.extend([1] * len(X_tr) + [0] * len(X_te))

            print(f"Shadow model {i+1}/{self.n_shadow} trained. "
                  f"Accuracy={shadow.score(X_te, y_te):.3f}")

        F = np.vstack(all_features)
        L = np.array(all_labels)

        self.attack_classifier.fit(F, L)
        self._trained = True

    def infer_membership(self, target_model, query_points):
        """
        给定一个训练好的目标模型和查询数据点，
        返回每个点的成员概率。
        1.0 = 可能是成员，0.0 = 可能不是成员。
        """
        if not self._trained:
            raise RuntimeError("请先调用 train_attack()。")

        features = self._extract_features(target_model, query_points)
        return self.attack_classifier.predict_proba(features)[:, 1]

    def evaluate(self, target_model, member_X, nonmember_X):
        """计算针对目标模型的攻击 AUC。"""
        member_scores    = self.infer_membership(target_model, member_X)
        nonmember_scores = self.infer_membership(target_model, nonmember_X)

        y_true  = np.concatenate([np.ones(len(member_X)), np.zeros(len(nonmember_X))])
        y_score = np.concatenate([member_scores, nonmember_scores])

        auc = roc_auc_score(y_true, y_score)
        print(f"Membership Inference AUC: {auc:.4f}")
        return auc

隐私影响：GDPR 与 HIPAA

成员推理直接违反了主要数据保护法律所载明的隐私原则。 根据GDPR 第 17 条（被遗忘权）， 个人可以请求删除其数据 — 但如果已部署的模型揭示了 成员身份，则除非重新训练或应用机器遗忘技术， 否则无法实现有效删除。根据HIPAA，用于训练模型 而未经过脱敏处理的健康信息，如果可以从已部署模型中推断出成员身份， 可能会使机构面临责任。监管机构越来越多地 将可证明的成员推理漏洞视为合规失败， 而不仅仅是理论风险。

05

属性推理

成员推理是一个二元问题：这个人是否在训练集中？ 属性推理更加精细：已知一个人 在训练集中（或仅仅是模型的输入），攻击者能否了解 其未作为输入明确提供的敏感属性？ 这类攻击尤其有害，因为它可以在查询时运作， 而不仅仅是在训练时 — 任何预测 API 都可能泄露 查询对象的人口统计或行为属性。

属性推理如何工作

考虑一个信用评分模型，其训练数据包括"批准的 特征"（收入、信用历史、贷款金额）和"受保护的属性"（种族、 性别、邮政编码作为种族的代理）。即使在推理时受保护属性被排除 在模型的官方特征集之外，模型也可能在训练期间吸收了 相关性。知道目标个体已批准特征 的攻击者可以查询模型，并通过观察预测来推断该个体的 受保护属性。研究表明，即使使用显式公平性约束训练的模型 仍然可以通过其输出分布泄露受保护属性。

攻击通常通过训练重建模型来工作： 攻击者收集许多（已知特征，模型输出）对，其中敏感属性 也是已知的（来自单独的数据集或通过相关性），并训练一个 分类器从模型输出预测敏感属性。Yeom et al. (2018) 将其形式化为一种攻击，每当模型学会 利用敏感属性和标签之间的相关性时，该攻击就会成功。

人口统计和行为推理

LLM 为属性推理提供了特别丰富的攻击面，因为它们产生 细微、开放式的输出。研究表明，LLM 可以仅通过 写作风格预测用户人口统计信息（年龄、性别、政治倾向、国籍）。 当 LLM 在用户交互日志上进行微调时，微调后的 模型可能通过其对查询做出响应的微妙系统性差异 暴露个体用户的特征 — 即使对于属于微调集 而非当前查询的用户也是如此。

交叉引用攻击

一种强大的变体将属性推理与辅助数据相结合。攻击者 不仅仅依赖模型；他们将模型输出与公开可用的 数据库、社交媒体资料和其他数据源相结合。例如，知道一个 医疗模型预测具有（年龄=54，邮政编码=90210，诊断=T2D）特征的 患者有 73% 的再入院风险 — 当与公开的 选民登记和财产记录交叉引用时 — 可能足以唯一识别该患者并 推断其完整的病史。

伦理和监管关切 属性推理攻击如果泄露用于重大决策（就业、信用、保险） 的受保护特征，可能构成非法歧视， 违反欧盟 AI 法案关于禁止 AI 实践的规定以及 CCPA 对敏感个人特征推理的保护。为重大决策 部署预测 API 的组织应将属性推理审计作为 模型发布流程的一部分。

缓解策略

防御属性推理需要训练时和部署时的 干预。面向公平性的对抗训练惩罚允许 判别器从中间表示推断受保护属性的模型。 输出限制 — 仅返回硬标签而非置信度 分数 — 减少了攻击者可用的信息，但并未消除 风险。带差分隐私的联邦学习可以限制 编码在模型权重中的个人级信息量，提供了最强 的理论保证。

06

针对 LLM 的侧信道攻击：Whisper Leak

LLM 部署中的一个普遍假设是 TLS/HTTPS 加密为用户查询提供了 有意义的保密性。Whisper Leak 研究 (2025) 打破了这一假设。该攻击表明，具有 对用户加密网络流量被动访问权限的攻击者 — 如互联网服务 提供商、被入侵的路由器或恶意 Wi-Fi 接入点 — 可以在不解密单个负载字节的情况下， 以超过 98% 的准确率推断用户 LLM 查询的主题， 在 28 个测试的商业 LLM 中有 17 个达到此水平。 [Whisper Leak, arXiv 2511.03675, 2025]

攻击前提：为何加密不够

TLS（使用 AES-GCM 等流密码）加密负载内容，但保留了 负载大小：size(ciphertext) = size(plaintext) + constant。 当 LLM 逐个令牌生成流式响应时，每个令牌作为 单独的加密数据包发送。由于令牌具有可变长度（令牌 the 是 3 个字符；antidisestablishmentarianism 是 28 个）， 数据包大小的序列直接编码了令牌长度的序列。

关键洞察在于不同的主题会产生系统性地不同的 令牌长度模式。关于量子物理的响应使用更长、更不 频繁的技术术语。关于烹饪的响应使用更短、更常见的词汇。 关于法律事务的响应包括特定的法律术语和拉丁短语。 这些模式在不同的用户提出类似问题时足够稳定， 以至于训练好的分类器可以仅从数据包大小识别主题 — 即使 看不到内容。

实验设置和结果

研究人员为每个模型收集了 21,716 个查询（100 个主题变体 × 100 次重复 + 11,716 个 Quora 噪声查询），并使用 tcpdump 捕获的 加密流量痕迹训练二元分类器。测试针对来自主要提供商的 28 个 LLM 进行，包括 OpenAI（GPT-4o-mini, GPT-4.1）、Anthropic（Claude 3 Haiku）、Google （Gemini 1.5 Flash, 2.5 Pro）、Microsoft、xAI（Grok）、Mistral、DeepSeek、Meta （通过 Lambda 访问的 LLaMA）和 Amazon（Nova）。 [Whisper Leak]

AUPRC（中位数）

28 个模型中的 17 个超过 98%。所有模型中平均 96.8%。

精确率

17/28 个模型在 5–20% 召回率下达到 100% 精确率（如 GPT-4o-mini, Mistral, Grok）。

最难目标

Google Gemini（81–84% AUPRC）和 Amazon Nova（71–77%）最具有抵抗性。

攻击者模型

被动网络观察者（ISP 级别）。无需主动干扰。

三种分类器架构

论文在数据包大小 + 到达间隔时间特征序列上评估了 三种分类器架构：

  • LightGBM：基于扁平化零填充数据包大小/时序序列 （填充到第 95 百分位长度）的梯度提升决策树集成。训练快速， 具有竞争力的准确率。
  • BiLSTM：带注意力机制的双向 LSTM。嵌入每个 （数据包大小，到达间隔时间）对，通过 BiLSTM + 注意力处理， 然后通过两层 MLP 头部进行分类。捕获序列依赖关系。
  • 基于 BERT（DistilBERT）：将数据包大小和时间离散化为 50 个分箱的令牌， 然后在这些令牌序列上微调 DistilBERT 分类器。 在具有复杂数据包分布的模型上表现最佳。

概念攻击代码：流量捕获和分类

import subprocess
import struct
import numpy as np
from collections import namedtuple
import joblib   # 用于加载预训练的 LightGBM 分类器


PacketTrace = namedtuple("PacketTrace", ["sizes", "timings"])


def capture_llm_traffic(target_host, duration_sec=30, interface="eth0"):
    """
    被动捕获发往/来自 LLM API 端点的加密 HTTPS 数据包。
    返回包含每个数据包大小和到达间隔时间的 PacketTrace。

    重要提示：仅在您获得授权的网络/系统上使用。
    这是对 Whisper Leak 方法论的概念性演示。

    使用 tcpdump（需要适当的权限或数据包捕获能力）。
    """
    # 在生产环境中，Whisper Leak 使用 Scapy 解析 tcpdump 输出
    # 此处展示结构；实际实现需要 pcap 解析

    cmd = [
        "tcpdump", "-i", interface,
        "-nn", "-q", "-tt",             # 时间戳 + 大小，无 DNS 解析
        f"host {target_host} and port 443",
        "-c", "2000"                       # 捕获最多 2000 个数据包
    ]

    # 解析格式如 "1700000000.123456 IP src > dst: ... length NNN" 的输出行
    raw_lines = subprocess.check_output(cmd, stderr=subprocess.DEVNULL).decode()

    sizes   = []
    timestamps = []
    prev_ts = None
    inter_arrivals = []

    for line in raw_lines.splitlines():
        parts = line.split()
        if not parts: continue
        try:
            ts  = float(parts[0])
            # 长度通常是最后一个以数字开头的令牌
            pkt_len = int([p for p in parts if p.isdigit()][-1])
            sizes.append(pkt_len)
            if prev_ts is not None:
                inter_arrivals.append(ts - prev_ts)
            prev_ts = ts
        except (ValueError, IndexError):
            continue

    return PacketTrace(sizes=sizes, inter_arrivals=inter_arrivals)


def featurize_trace(trace, max_len=512):
    """
    将数据包大小和到达间隔序列填充/截断到固定长度，
    然后拼接成用于 LightGBM 分类的特征向量。
    镜像 Whisper Leak 的特征化策略。
    """
    sizes   = np.array(trace.sizes[:max_len],   dtype=np.float32)
    timings = np.array(trace.inter_arrivals[:max_len], dtype=np.float32)

    # 零填充到 max_len
    sizes_padded   = np.pad(sizes,   (0, max_len - len(sizes)))
    timings_padded = np.pad(timings, (0, max_len - len(timings)))

    # 将大小和时间拼接成一个特征向量
    feature_vector = np.concatenate([sizes_padded, timings_padded])
    return feature_vector.reshape(1, -1)


def classify_prompt_topic(trace, classifier_path, topic_labels):
    """
    给定捕获的流量痕迹，加载预训练的 LightGBM 分类器
    并预测底层 LLM 提示的主题。
    """
    clf = joblib.load(classifier_path)      # 预训练的二元或多类分类模型
    features = featurize_trace(trace)
    proba = clf.predict_proba(features)[0]

    results = sorted(zip(topic_labels, proba), key=lambda x: x[1], reverse=True)
    print("主题分类结果：")
    for topic, prob in results[:5]:
        print(f"  {topic:30} {prob:.3f}")
    return results[0][0]  # 预测概率最高的主题

防御措施

Whisper Leak 论文评估了随机填充作为缓解措施： 向每个流式令牌添加随机长度的字符串以隐藏其长度。 Cloudflare 在初始披露后实施了此防御。然而，论文表明 即使有填充，数据包之间的时序信息仍保留 显著的可分类信号。 唯一完全有效的防御是 恒定形状流量：将所有响应填充到固定大小并在传输前 批量处理令牌，消除所有数据包大小变化。 这从根本上与大多数 LLM 提供商优化的低延迟流式用户体验相冲突。

07

令牌长度侧信道

Whisper Leak 攻击推断提示的主题。而 Weiss et al. (2024) 的研究追求一个更精确的目标： 从加密数据包的大小推断 LLM 响应的确切内容 — 逐字逐句。这就是令牌长度侧信道， 它代表了 LLM 部署历史上最令人震惊的隐私失败之一。 [Microsoft Security Blog, 2025]

攻击方法论

当 LLM 逐个令牌流式传输其响应时，每个 TLS 数据包正好携带 一个令牌的字节。数据包大小直接揭示了令牌的字节长度： 3 字节数据包携带 3 字符令牌；7 字节数据包携带 7 字符 令牌。因此，观察加密流的攻击者可以了解到响应中每个令牌的 长度序列 — 例如 [4, 5, 1, 3, 6, 2, 7, ...]。

有了这个长度序列，Weiss et al. 使用第二个 LLM 来重建 与该精确令牌长度模式匹配的最可能句子。该任务本质上是 一个受限文本生成问题：生成一个连贯的句子， 其分词结果恰好产生观察到的长度序列。鉴于 分词器的词汇表是固定的且广为人知（例如 OpenAI 模型的 tiktoken）， 这是一个极度受限的问题 — 远比无约束生成容易。

该攻击实现了约 27% 的精确令牌重建。 虽然这看起来不算高，但请考虑即使部分重建 也足以确定响应是否包含敏感的 医疗诊断、法律建议或私人个人信息。

为何此攻击在结构上难以修复

根本原因不是 TLS 的缺陷 — 而是流式 API 设计的 基本属性。任何系统只要：

  1. 使用固定的分词器（因此令牌长度可从词汇表预测），
  2. 通过 TLS 逐一流式传输令牌，且
  3. 使用保留明文长度的流密码，

都容易受到此攻击。唯一能完全消除它的缓解措施是： (a) 向所有令牌添加确定性填充使它们看起来大小相同， 或 (b) 在加密前将多个令牌批量打包到每个数据包中 — 这两种方法 都会增加延迟并降低使流式传输有价值的响应性。 Cloudflare 在 Weiss et al. 披露后在 CDN 层实现了每令牌随机填充， 但后续工作仍然展示了残余的信息泄露。 [Whisper Leak citing Cloudflare mitigation]

对敏感部署的影响

任何部署在医疗、法律、金融或政府环境中且使用流式 API 的 LLM， 都可能向用户与提供商之间具有网络可见性的任何人 泄露响应内容。这包括企业代理、 VPN 提供商、ISP 以及潜在的政府监控基础设施。 对于高度敏感部署的适当安全姿态是彻底禁用流式传输 （将完整响应作为单个数据包返回）或 在没有外部网络暴露的情况下在本地部署 LLM 推理。

08

针对高效推理的时序攻击

性能在 LLM 服务市场中具有竞争性，提供商大力投资于 推理优化技术。两大类别 — 推测性解码 和 KV 缓存共享 — 已被证明会引入 可利用的侧信道，泄露用户输入和系统配置的 私有信息。

推测性解码侧信道

推测性解码通过让一个小型、廉价的"草稿模型" 提前预测几个令牌，然后让大型目标模型并行验证它们来加速 LLM 推理。 当草稿模型猜对时，多个令牌在一次验证传递中被接受。 当它猜错时，模型回退到单令牌自回归 生成。这种接受/拒绝模式是输入相关的：某些 提示将被解码出许多正确的推测（每次迭代产生更大的 数据包），而其他提示会触发许多错误推测（每次迭代产生更小的数据包）。

Wei et al. (2024) 证明，一个观察每次生成迭代数据包大小的 被动网络对手可以重建这种接受/拒绝痕迹，并使用它 以 超过 90% 的准确率 对四种不同的推测性解码方案 中的用户查询进行指纹识别。 [Wei et al., "When Speculation Spills Secrets," 2024] 具体来说：REST 实现了约 100% 的查询识别准确率，LADE 达到 92%， BiLD 达到 95%，甚至在远程 vLLM 上的 EAGLE 也达到了 77.6% 的准确率。

除了查询指纹识别，具有 API 访问权限的恶意用户还可以提取 私有数据存储内容，这些内容被检索增强型推测性解码使用 （例如 REST）：通过精心构造用于探测数据存储的输入并观察哪些 令牌被正确推测，攻击者可以以每秒超过 25 个令牌的速度 泄露数据存储内容。这对于在检索语料库中包含 专有或机密文档的 RAG 系统来说是一个特别严重的威胁。

Carlini & Nasr：推理优化的时序变化

Carlini & Nasr (2024) 展示了此类攻击的早期版本， 表明商业模型（GPT-4, Claude）中推理优化导致的时序变化 可以通过数据包到达间隔时间作为侧信道加以利用。 他们的工作为这一研究领域建立了威胁模型，尽管后续 研究表明到达间隔时间信号比数据包大小信号 嘈杂得多 — Wei et al. 的方法在相同的 vLLM 设置上达到 77.6% 的准确率， 而 Carlini & Nasr 的方法仅达到 14.4%。

InputSnatch：KV 缓存时序攻击

第二类时序侧信道利用了 KV 缓存共享。 现代 LLM 推理后端（vLLM, TensorRT-LLM）实现了前缀缓存：如果两个 请求共享一个公共前缀，则为该前缀计算的 KV 缓存会被重用 而非重新计算。这产生了可测量的时序差异 — 缓存命中的响应 明显快于缓存未命中。 [Zheng et al., "InputSnatch," arXiv 2411.18191, 2024]

Zheng et al. (2024) 的 InputSnatch 攻击利用此 漏洞重建其他用户缓存的提示。该攻击通过系统性地 使用候选输入查询目标服务并观察 首令牌时间（TTFT）是否指示缓存命中来工作。缓存命中表明 候选前缀与另一个用户的缓存查询匹配。通过迭代 构建越来越长的匹配前缀，攻击者可以重建 受害者用户的精确提示 — 即使服务使用 HTTPS 加密。

在针对具有前缀缓存的医疗问答聊天机器人的实验中，InputSnatch 实现了 62% 的成功率来提取确切的疾病输入，以及 13.5% 用于 精确症状描述。对于具有语义缓存的法律咨询 RAG 系统， 语义提取成功率范围为 43% 到 100%。

InputSnatch 攻击流程： [攻击者] ─── 构造候选前缀 "What are the symptoms of..." ──→ [LLM API] │ 测量 TTFT │ ┌─────────────────────────────────────┘ │ TTFT < 阈值？ ──→ 是 → 缓存命中 → 前缀与受害者查询匹配 │ 否 │ TTFT ≥ 阈值 → 缓存未命中 → 尝试下一个候选 │ [使用 ML 引导的输入构造器迭代] │ 从匹配的前缀片段重建完整受害者查询

TPUXtract：硬件辐射攻击

到目前为止讨论的所有攻击都运行在 API 或网络层面。但还有 一个完全不同的攻击面： 运行模型的物理硬件。专用芯片 （TPU、GPU、NPU）上的神经网络推理会产生与正在执行的操作 相关的电磁（EM）辐射。对硬件具有物理接近度的攻击者 — 甚至在数据中心相邻机架上接触到测量设备的攻击者 — 可以利用这些辐射来推断模型的架构。这就是 TPUXtract。 [Keysight Security Blog, 2025]

攻击方法论

该攻击由北卡罗来纳州立大学的研究人员针对 Google Tensor Processing Unit (TPU) 演示。 基本观察是 TPU 的功耗根据正在处理的层配置 发生可测量的变化。不同的层类型（卷积、全连接、注意力）、不同的层大小 以及不同的连接模式各自产生独特的 EM 特征。

TPUXtract 利用了这样一个事实：在神经网络中，数据顺序 流经各层。攻击者测量 TPU 处理输入时 随时间变化的 EM 信号， 并将不同的时间窗口与预期针对不同层配置的 EM 特征相关联。 通过将观察到的 EM 痕迹与预表征的层特征库进行匹配， 攻击者逐层重建模型的架构。

这种逐层方法与试图一次性匹配整个模型相比， 大大降低了搜索复杂度。该攻击在从 TPU 提取神经网络超参数方面 达到了 99.91% 的准确率。

可以提取什么

TPUXtract 可以恢复：

  • 层数 — 网络的深度
  • 层类型 — 全连接、卷积、注意力、归一化
  • 层维度 — 每层的神经元/通道数
  • 连接模式 — 跳跃连接、注意力头

关键的是，TPUXtract 不恢复模型权重 — 即训练期间学习的 实际数值参数。这是一个重要的限制： 知道架构就像知道建筑的蓝图而不知道 内部的家具。然而，知道架构具有巨大的价值： 它使得通过 API 进行更高效的模型提取成为可能（代理模型的搜索空间 现在被极大地约束），它揭示了专有的 架构创新，并为针对性的对抗攻击提供了路线图。 对于基于 Transformer 的 LLM，完整提取需要额外的步骤 （消零特定层的权重以隔离其他层），但论文 证明了可行性。

对模型知识产权安全的影响

云提供商和硬件制造商历来假设加速器的 EM 辐射 在多租户环境中不可利用， 因为租户之间的隔离应阻止物理访问。TPUXtract 挑战了这一 假设：在共置数据中心，相邻物理硬件上的测量可能 就足够了。更广泛地说，任何在未完全端到端物理控制的硬件上 运行 AI 推理的组织都面临架构保密性风险。 有效的对策包括硬件级 EM 屏蔽、噪声注入 电路和功耗掩盖 — 所有这些都来自密码硬件安全领域的 成熟技术，现在变得与 AI 部署相关。 [Dark Reading, 2024]

模型反演

模型反演攻击从模型输出重建有代表性的输入， 实质上是在反向运行推理。攻击者不问"这个模型对 这个输入预测什么？"，而是问"这个模型将什么输入与 这个预测相关联？"在人脸识别领域，模型反演可以 仅凭模型的置信度分数重建训练对象的可识别人脸图像 — 这是严重的隐私侵犯。

Fredrikson et al.：人脸重建

由 Fredrikson, Jha, and Ristenpart (CCS 2015) 撰写的开创性模型反演论文 证明，在一组具名个体上训练的人脸识别模型 可以被反演，为标签空间中任何目标个体 生成可识别的人脸图像。 [Fredrikson et al., CCS 2015] 该攻击通过在输入空间中进行基于梯度的优化来工作：从 随机噪声开始，攻击者迭代调整输入以最大化模型对 目标标签的置信度。 优化收敛到一个高度可识别的 目标个体图像 — 即使从未在训练集中看到其真实照片。

这种攻击之所以有效，是因为模型在其参数中编码了关于每个 个体面部特征的足够信息以做出自信的预测 — 而基于梯度的反演可以将这些信息解码回图像空间。 该攻击成功地生成了人脸图像，人类评估者能够以 显著高于随机水平的正确率将其识别为目标个体。

基于梯度的反演代码

import torch
import torch.nn as nn
import torch.optim as optim
import numpy as np


def model_inversion_attack(
    model,
    target_class,
    input_shape,
    n_iterations=2000,
    lr=0.01,
    reg_strength=0.001
):
    """
    白盒模型反演：重建一个最大程度激活模型输出中
    target_class 的输入。

    参数：
        model         : PyTorch 分类模型（应输出 logits 或 log-probs）
        target_class  : int，要反演的类别索引
        input_shape   : tuple，例如单张 RGB 64×64 图像为 (1, 3, 64, 64)
        n_iterations  : 优化步数
        lr            : 学习率
        reg_strength  : 输入上的 L2 正则化（防止退化解）

    返回：
        reconstructed_input : 形状为 input_shape 的 torch.Tensor
    """
    model.eval()

    # 从 [0, 1] 范围内的随机噪声初始化
    x = torch.rand(input_shape, requires_grad=True)
    optimizer = optim.Adam([x], lr=lr)
    criterion = nn.CrossEntropyLoss()

    target = torch.tensor([target_class], dtype=torch.long)

    for step in range(n_iterations):
        optimizer.zero_grad()

        # Clamp input to valid image range
        x_clamped = torch.clamp(x, 0.0, 1.0)

        # 通过目标模型进行前向传播
        logits = model(x_clamped)

        # 损失：最大化 target_class 的置信度 + 正则化以保持自然
        classification_loss = criterion(logits, target)
        regularization      = reg_strength * torch.norm(x_clamped, p=2)
        total_loss = classification_loss + regularization

        total_loss.backward()
        optimizer.step()

        if step % 500 == 0:
            confidence = torch.softmax(logits, dim=-1)[0, target_class].item()
            print(f"Step {step:4d} | Loss={total_loss.item():.4f} | "
                  f"Confidence for class {target_class}: {confidence:.3f}")

    return x.detach().clamp(0, 1)


def black_box_inversion(model_query_fn, target_class, input_shape,
                         n_iterations=5000, population_size=50):
    """
    使用自然进化策略（NES）的黑盒模型反演。
    model_query_fn: 接受输入数组并返回置信度分数的可调用对象。
    使用来自随机方向分数差的估计梯度。
    """
    sigma = 0.1    # 梯度估计的噪声尺度
    lr    = 0.01   # 学习率

    # 从均匀分布的均值开始
    x = np.random.uniform(0, 1, input_shape).astype(np.float32)

    for step in range(n_iterations):
        # 通过随机扰动估计梯度
        noise = np.random.randn(population_size, *input_shape).astype(np.float32)
        rewards = np.zeros(population_size)

        for i, n in enumerate(noise):
            x_perturbed = np.clip(x + sigma * n, 0, 1)
            scores = model_query_fn(x_perturbed)
            rewards[i] = scores[target_class]   # 最大化目标类别置信度

        # NES 梯度估计
        grad_estimate = np.mean(
            rewards[:, None, ...] * noise.reshape(population_size, -1),
            axis=0
        ).reshape(input_shape) / sigma

        x = np.clip(x + lr * grad_estimate, 0, 1)

        if step % 1000 == 0:
            curr_confidence = model_query_fn(x)[target_class]
            print(f"Step {step} | Target confidence: {curr_confidence:.3f}")

    return x

现代模型反演攻击通过利用生成对抗网络（GAN） 和扩散模型作为 输入分布的先验知识，已经变得极其强大。基于 GAN 的反演将 搜索限制在同一领域训练的 GAN 的潜在空间中，确保 重建图像具有照片级真实感和语义有效性。这种方法 已经实现了 64×64 分辨率的人脸重建，即使针对生产级人脸识别模型， 人类评估者也能识别出来。

防御：速率限制

对抗模型提取最简单、最可立即部署的防御是 速率限制：限制任何单个用户或 IP 地址 在单位时间内可以进行的查询数量。由于模型提取需要 数千到数十万次 API 调用，恰当的速率限制 会极大地增加攻击的时间和金钱成本，可能 使其在经济上不可行。

自适应速率限制

静态速率限制是一种生硬的工具 — 它们可能阻止合法的重度用户， 而老练的攻击者将其查询分布在许多账户或 IP 地址之间。自适应速率限制监控 区分合法使用与提取尝试的行为信号并动态调整 限制。提取查询往往是系统性的（有规律间隔、以结构化方式 覆盖输入空间），而合法查询往往是不规则的 且语义连贯。每用户查询预算、查询分布上的异常检测 以及渐进式限流构成了分层防御。

实现

import time
import collections
import numpy as np
from dataclasses import dataclass, field
from typing import Dict, Deque


@dataclass
class UserQueryRecord:
    """用于提取检测的每用户状态。"""
    query_times:    Deque[float] = field(default_factory=lambda: collections.deque(maxlen=1000))
    query_count:    int = 0
    flagged_count:  int = 0
    throttle_until: float = 0.0    # 限流过期的 Unix 时间戳


class AdaptiveRateLimiter:
    """
    检测类似提取的查询模式的自适应速率限制器。
    实现：
      1. 硬性每分钟速率限制
      2. 每用户每日预算
      3. 规律性异常检测（提取查询过于规律）
      4. 对可疑用户进行渐进式限流
    """

    def __init__(
        self,
        hard_limit_per_minute=60,
        daily_budget=5000,
        regularity_threshold=0.15,    # 低于此变异系数 = 可疑
        throttle_multiplier=4.0        # 按此因子减慢可疑用户速度
    ):
        self.hard_limit_per_minute = hard_limit_per_minute
        self.daily_budget          = daily_budget
        self.regularity_threshold  = regularity_threshold
        self.throttle_multiplier   = throttle_multiplier

        self.users: Dict[str, UserQueryRecord] = {}

    def _get_user(self, user_id: str) -> UserQueryRecord:
        if user_id not in self.users:
            self.users[user_id] = UserQueryRecord()
        return self.users[user_id]

    def _queries_last_minute(self, record: UserQueryRecord) -> int:
        now = time.time()
        cutoff = now - 60
        return sum(1 for t in record.query_times if t > cutoff)

    def _is_too_regular(self, record: UserQueryRecord) -> bool:
        """
        提取机器人倾向于以非常规律的间隔进行查询。
        计算查询间隔的变异系数（CV）。
        低 CV（规律）→ 标记为可疑。
        """
        times = list(record.query_times)
        if len(times) < 20:
            return False   # 数据不足
        intervals = np.diff(sorted(times[-50:]))   # 最近 50 次查询
        if len(intervals) == 0 or np.mean(intervals) == 0:
            return False
        cv = np.std(intervals) / np.mean(intervals)
        return cv < self.regularity_threshold

    def check_and_record(self, user_id: str) -> dict:
        """
        检查是否应允许来自 user_id 的查询。
        返回 {"allowed": bool, "reason": str, "throttle_remaining": float}
        如果允许，则记录查询时间。
        """
        record = self._get_user(user_id)
        now    = time.time()

        # 1. 检查用户是否当前处于活跃限流状态
        if now < record.throttle_until:
            return {
                "allowed": False,
                "reason": "throttled",
                "throttle_remaining": record.throttle_until - now
            }

        # 2. 检查硬性每分钟速率限制
        if self._queries_last_minute(record) >= self.hard_limit_per_minute:
            return {
                "allowed": False,
                "reason": "rate_limit_exceeded",
                "throttle_remaining": 0
            }

        # 3. 检查每日预算
        if record.query_count >= self.daily_budget:
            return {
                "allowed": False,
                "reason": "daily_budget_exceeded",
                "throttle_remaining": 0
            }

        # 4. 检查类似提取的规律性
        suspicious = self._is_too_regular(record)
        if suspicious:
            record.flagged_count += 1
            # 渐进式：每次标记使限流窗口翻倍
            throttle_seconds = min(60 * self.throttle_multiplier ** record.flagged_count, 86400)
            record.throttle_until = now + throttle_seconds
            return {
                "allowed": False,
                "reason": "extraction_pattern_detected",
                "throttle_remaining": throttle_seconds
            }

        # 5. 允许：记录查询
        record.query_times.append(now)
        record.query_count += 1
        return {"allowed": True, "reason": "ok", "throttle_remaining": 0}
纵深防御 仅靠速率限制不足以应对耐心且分布式的攻击者。将其 与输出扰动（第 12 节）和水印（第 13 节）结合以形成分层 防御。通过速率限制的攻击者仍应收到扰动后的 输出，并且任何产生的代理模型应携带可检测的水印。

防御：输出扰动

即使攻击者成功收集了数千个查询-响应对，其 代理模型的保真度仍取决于这些标签的质量。如果 目标模型的输出被扰动 — 通过添加校准噪声、舍入 置信度分数或限制输出为前 k 个类别 — 代理模型 将在被破坏的监督信号上训练并降低质量。输出扰动的 艺术在于添加足够的噪声以阻碍提取，同时保留 足够的信号以维持对合法用户的实用性。

输出的差分隐私机制

差分隐私理论中的拉普拉斯机制和高斯机制 提供了添加输出噪声的原则性方法， 具有形式化保证。对于返回 [0,1]^k 中概率向量的模型， 添加尺度为 Δf / ε（其中 Δf 是 输出函数的 L1 敏感度，ε 是隐私参数）的拉普拉斯噪声 确保 (ε, 0)-差分隐私。在实践中，softmax 输出的敏感度 为 2（有界范围），因此噪声尺度为 2/ε。

置信度分数舍入和 Top-k 限制

一种更简单、非概率的方法是置信度分数舍入： 返回舍入到 2 位小数的概率而不是 8 位。这极大地 减少了每个查询响应的信息内容，同时保留了类别的 序数排名（这是大多数合法用户所需要的）。Top-k 限制仅返回前 k 个预测而不是完整的分布， 进一步限制了代理模型可用的信息。

实现

import numpy as np
from scipy.special import softmax


class OutputPerturbationDefense:
    """
    通过在返回给用户之前扰动模型输出，
    防御模型提取。

    支持三种策略：
      - 'laplace'  : 添加拉普拉斯噪声（差分隐私）
      - 'rounding' : 将置信度舍入到 d 位小数
      - 'topk'     : 仅返回前 k 个预测
    """

    def __init__(self, strategy='laplace', epsilon=1.0, decimal_places=2, top_k=3):
        assert strategy in ('laplace', 'gaussian', 'rounding', 'topk')
        self.strategy       = strategy
        self.epsilon        = epsilon          # 隐私预算（越小 = 噪声越大）
        self.decimal_places = decimal_places
        self.top_k          = top_k

    def perturb(self, probabilities: np.ndarray) -> np.ndarray:
        """
        对概率向量应用输出扰动。
        probabilities: 形状为 (n_classes,) 的 np.ndarray，求和为 1。
        返回扰动后的概率（对于噪声方法不一定求和为 1）。
        """
        proba = np.asarray(probabilities, dtype=np.float64)

        if self.strategy == 'laplace':
            return self._laplace_mechanism(proba)
        elif self.strategy == 'gaussian':
            return self._gaussian_mechanism(proba)
        elif self.strategy == 'rounding':
            return self._rounding(proba)
        elif self.strategy == 'topk':
            return self._topk_restriction(proba)

    def _laplace_mechanism(self, proba):
        """
        添加尺度为 sensitivity / epsilon 的拉普拉斯噪声。
        对于概率向量，L1 敏感度为 2。
        噪声添加后，重新归一化并裁剪到 [0, 1]。
        """
        sensitivity = 2.0   # softmax 输出的 L1 敏感度
        scale = sensitivity / self.epsilon
        noise = np.random.laplace(loc=0.0, scale=scale, size=proba.shape)
        noisy = np.clip(proba + noise, 0.0, 1.0)
        # 重新归一化使总和为 1（投影到概率单纯形）
        total = noisy.sum()
        return noisy / total if total > 0 else np.ones_like(noisy) / len(noisy)

    def _gaussian_mechanism(self, proba):
        """
        具有 (epsilon, delta)-DP 保证的高斯机制。
        默认使用 delta=1e-5；sigma 根据 L2 敏感度校准。
        """
        delta = 1e-5
        l2_sensitivity = np.sqrt(2)           # 概率向量的 L2 敏感度
        sigma = l2_sensitivity * np.sqrt(2 * np.log(1.25 / delta)) / self.epsilon
        noise = np.random.normal(0, sigma, proba.shape)
        noisy = np.clip(proba + noise, 0.0, 1.0)
        total = noisy.sum()
        return noisy / total if total > 0 else np.ones_like(noisy) / len(noisy)

    def _rounding(self, proba):
        """将每个概率舍入到 d 位小数，然后重新归一化。"""
        rounded = np.round(proba, self.decimal_places)
        total   = rounded.sum()
        return rounded / total if total > 0 else np.ones_like(rounded) / len(rounded)

    def _topk_restriction(self, proba):
        """
        将除前 k 个类别之外的所有类别置零并重新归一化。
        返回最多有 k 个非零条目的稀疏向量。
        """
        k = min(self.top_k, len(proba))
        result = np.zeros_like(proba)
        top_indices = np.argsort(proba)[-k:]
        result[top_indices] = proba[top_indices]
        total = result.sum()
        return result / total if total > 0 else result

防御：模型水印

速率限制和输出扰动试图阻止提取。模型水印 采取了不同的方法：它假设提取可能发生， 并在模型行为中嵌入一个可验证的签名，该签名会持续 保留到代理模型中，允许原始模型所有者证明一个 可疑的被盗模型源自其源模型。 [Survey: IP Protection for Deep Learning, arXiv 2411.05051, 2024]

基于后门的水印

最广泛部署的水印技术引入一个秘密触发 集：一小部分精心制作的输入-输出对，模型被训练成 以一种特定的、不寻常的方式对其做出响应。例如，一个人脸 识别模型可能被加水印，以极高的置信度将包含特定 微妙纹理模式的图像分类为指定的"水印类别"。 模型的合法副本（包括通过在原始输出的知识蒸馏上 训练的任何代理模型）也会表现出这种行为，因为 攻击者的训练查询包含了这些触发输入，并且攻击者 忠实地复制了相应的响应。当模型所有者怀疑 存在被盗副本时，他们用触发集查询它并检查预期的 水印行为。

参数空间水印

白盒水印直接将签名嵌入模型权重而非 行为。Uchida et al. (2017) 提出使用训练期间的正则化项 将位字符串嵌入特定层权重值的分布中。 水印可以通过计算权重向量与密钥矩阵的内积 来提取。这种方法比基于后门的方法对模型修改 （微调、剪枝）更加鲁棒，但需要访问模型的内部权重 进行验证 — 这只有在攻击者提供其代理模型供检查时 才有可能。

局限性和活跃研究

水印面临几个根本性挑战。首先，知道水印存在的 坚定攻击者可以尝试通过微调、模型剪枝或将知识 蒸馏到一个全新模型中来移除它。其次，基于后门的 水印引入了真正的安全漏洞 — 如果触发模式 被发现，它可以被用来操纵模型的行为。第三， 水印提供了事后归因，但并不能阻止提取的初始 损害。尽管如此，出于知识产权诉讼的目的，一个能够 在提取后幸存并且与被受害者模型具有可证明的非平凡相关性的 鲁棒水印提供了强有力的盗窃法律证据。

最近关于放射性数据的工作 — 用不可察觉的扰动污染训练数据， 这些扰动会传播到在其上训练的模型中 — 提供了一种 替代性的水印方法，它在数据层面而非模型层面运作， 即使攻击者从被盗训练数据而非从 API 蒸馏中训练 一个全新的模型，也能提供归因。

针对提取尝试的 API 监控

一个检测工具完善的 API 可以通过监控行为异常来检测正在进行的模型提取。 合法 API 用户表现出特有的使用模式 （与特定用例相关的突发查询、NLP 应用中的自然语言多样性、 可预测的日间模式），这些模式与提取攻击不同 （对输入空间的系统性覆盖、高查询量、机械生成的 输入、低语义多样性）。有效的 API 监控将统计 异常检测与特定领域的提取启发式方法相结合。

提取行为特征

  • 不寻常的查询分布：提取查询倾向于均匀覆盖 输入域或沿特定信息论标准覆盖， 产生的输入分布与自然使用截然不同。
  • 高查询速率：来自单个账户或关联账户的 接近最大速率的 API 调用。
  • 低语义连贯性：对于 NLP 模型，提取查询 可能包含部分随机化的文本、边缘情况输入或语法 不寻常的结构，这些不会来自真实的用户需求。
  • 缺乏反馈模式：合法用户通常会对错误或低置信度响应 进行后续跟进。提取机器人通常不会。
  • 跨账户协调：多个账户具有相似的 查询模式或查询输入空间的互补区域。

import numpy as np
from sklearn.ensemble import IsolationForest
from sklearn.preprocessing import StandardScaler
import collections
import math


class ExtractionDetector:
    """
    基于异常的模型提取尝试检测器。
    从查询统计信息为每个用户会话构建特征向量，
    并使用孤立森林识别异常使用模式。
    """

    def __init__(self, window_size=100, contamination=0.05):
        """
        window_size   : 每用户要考虑的最近查询数
        contamination : 异常用户的预期比例（用于 IsolationForest）
        """
        self.window_size   = window_size
        self.detector      = IsolationForest(contamination=contamination, random_state=42)
        self.scaler        = StandardScaler()
        self.user_history  = collections.defaultdict(list)   # user_id -> [查询特征]
        self._fitted       = False

    def _compute_session_features(self, user_id: str) -> np.ndarray:
        """
        计算总结用户最近查询行为的特征向量。
        特征：
          0: queries_per_minute  （每分钟查询数，最近窗口内）
          1: inter_query_cv      （间隔的变异系数）
          2: input_entropy       （输入长度的多样性）
          3: confidence_mean     （平均模型置信度 — 对抗输入较低）
          4: confidence_cv       （模型置信度的变异）
          5: unique_input_ratio  （不同输入的比例 — 提取时较高）
        """
        history = self.user_history[user_id][-self.window_size:]
        n = len(history)
        if n < 2:
            return np.zeros(6)

        timestamps      = np.array([h["timestamp"]      for h in history])
        input_lengths   = np.array([h["input_length"]    for h in history])
        confidences     = np.array([h["top_confidence"]  for h in history])
        input_hashes    =          [h["input_hash"]      for h in history]

        intervals = np.diff(sorted(timestamps))

        # 特征 0：查询速率（观察窗口内每分钟查询数）
        time_span = timestamps.max() - timestamps.min()
        qpm = (n / time_span * 60) if time_span > 0 else 0

        # 特征 1：查询间隔规律性（提取机器人 → 低 CV）
        cv_intervals = (np.std(intervals) / np.mean(intervals)) if np.mean(intervals) > 0 else 0

        # 特征 2：输入长度的熵（系统性扫描 → 低熵）
        length_counts = collections.Counter(input_lengths)
        total = sum(length_counts.values())
        entropy = -sum((c/total) * math.log2(c/total + 1e-9) for c in length_counts.values())

        # 特征 3 & 4：模型置信度统计
        conf_mean = np.mean(confidences)
        conf_cv   = np.std(confidences) / conf_mean if conf_mean > 0 else 0

        # 特征 5：唯一输入比例（系统性提取时接近 1.0）
        unique_ratio = len(set(input_hashes)) / n

        return np.array([qpm, cv_intervals, entropy, conf_mean, conf_cv, unique_ratio])

    def record_query(self, user_id: str, timestamp: float, input_length: int,
                       top_confidence: float, input_hash: str):
        """记录单个 API 查询的元数据（为保护隐私不存储原始输入）。"""
        self.user_history[user_id].append({
            "timestamp":      timestamp,
            "input_length":   input_length,
            "top_confidence": top_confidence,
            "input_hash":     input_hash,
        })

    def fit_baseline(self, baseline_user_ids):
        """在基线合法用户会话上训练异常检测器。"""
        features = [self._compute_session_features(uid) for uid in baseline_user_ids
                    if len(self.user_history[uid]) >= 10]
        if not features:
            raise ValueError("No baseline data available.")
        X = np.array(features)
        X_scaled = self.scaler.fit_transform(X)
        self.detector.fit(X_scaled)
        self._fitted = True

    def score_user(self, user_id: str) -> float:
        """
        返回用户的异常分数。
        负分数 → 异常（潜在的提取行为）。
        正分数 → 正常行为。
        """
        if not self._fitted:
            raise RuntimeError("Call fit_baseline() first.")
        features = self._compute_session_features(user_id).reshape(1, -1)
        scaled   = self.scaler.transform(features)
        return self.detector.decision_function(scaled)[0]

    def is_suspicious(self, user_id: str, threshold: float = 0.0) -> bool:
        """如果用户的行为异常程度超过阈值，返回 True。"""
        return self.score_user(user_id) < threshold

差分隐私

到目前为止讨论的所有防御措施都在推理时运作 — 即模型训练完成之后。 差分隐私（DP）通过修改训练过程本身来解决根本原因， 限制任何单个训练样本对最终模型的影响程度。差分隐私模型被形式化地保证 只揭示关于任何单个训练样本的有界信息 — 提供 对成员推理、属性推理和训练数据提取攻击的 可证明的抵抗力。

形式化定义

一个随机化机制 M 是 (ε, δ)-差分隐私的， 如果对于任意两个相邻数据集 D 和 D'（恰好相差 一条记录），以及对于任意输出集合 S：

Pr[M(D) ∈ S] ≤ eε · Pr[M(D') ∈ S] + δ

更小的 ε 意味着更强的隐私性。当 δ = 0（纯 DP）时，保证是绝对的； 允许 δ > 0（近似 DP，通常 δ < 1/n）允许略微放松 保证以换取显著更好的效用。隐私预算 ε 跟踪该机制所有使用中的总信息泄露 — 它随着每次 查询、访问或训练步骤而消耗，提供了一个随时间管理隐私风险的 量化框架。

DP-SGD：使用差分隐私训练

差分隐私随机梯度下降（DP-SGD）， 由 Abadi et al. (2016) 提出，是用差分隐私训练神经网络 的标准机制。该算法以两种方式修改标准 SGD：

  1. 梯度裁剪：每个样本的梯度被单独计算 （而不是在批次上平均），并裁剪到最大 L2 范数 C。这限制了 任何单个训练样本对梯度更新的影响。
  2. 高斯噪声添加：在参数更新之前， 将标准差与 C × σ 成比例的 高斯噪声添加到裁剪后的梯度求和中，其中 σ 是根据所需 (ε, δ) 预算校准的噪声乘数。

DP-SGD 的代价是模型准确率的降低：更多噪声意味着梯度信号 效果较差，尤其是在训练早期。隐私-效用权衡由 ε 控制： ε < 1 的值提供强隐私性但显著的准确率损失； ε = 1–10 提供中等隐私性，准确率成本适中（通常在分类基准上 为 2–5%）。

import torch
import torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset
from opacus import PrivacyEngine
from opacus.utils.batch_memory_manager import BatchMemoryManager
import numpy as np


def train_with_dp(
    model:           nn.Module,
    train_loader:    DataLoader,
    optimizer:       torch.optim.Optimizer,
    n_epochs:        int,
    target_epsilon:  float = 1.0,   # 隐私预算
    target_delta:    float = 1e-5,  # 失败概率
    max_grad_norm:   float = 1.0,   # 梯度裁剪范数
    noise_multiplier:float = 1.1,   # σ: 越大 → 越隐私，越少效用
    device:          str   = "cpu"
) -> dict:
    """
    使用 Opacus 库 (meta-pytorch/opacus) 以 (target_epsilon, target_delta)-差分隐私
    训练 PyTorch 模型。

    返回：包含最终 epsilon、delta 和每 epoch 损失的字典。
    """
    model = model.to(device)
    criterion = nn.CrossEntropyLoss()

    # 附加 PrivacyEngine 以在训练期间强制执行 DP
    privacy_engine = PrivacyEngine()

    model, optimizer, train_loader = privacy_engine.make_private_with_epsilon(
        module=model,
        optimizer=optimizer,
        data_loader=train_loader,
        epochs=n_epochs,
        target_epsilon=target_epsilon,
        target_delta=target_delta,
        max_grad_norm=max_grad_norm,
    )
    # 等效于直接指定 noise_multiplier：
    # model, optimizer, train_loader = privacy_engine.make_private(
    #     module=model, optimizer=optimizer, data_loader=train_loader,
    #     noise_multiplier=noise_multiplier, max_grad_norm=max_grad_norm,
    # )

    history = []
    for epoch in range(1, n_epochs + 1):
        model.train()
        epoch_losses = []

        with BatchMemoryManager(
            data_loader=train_loader,
            max_physical_batch_size=64,   # 针对 DP 的内存高效批处理
            optimizer=optimizer
        ) as memory_safe_loader:
            for data, target in memory_safe_loader:
                data, target = data.to(device), target.to(device)
                optimizer.zero_grad()
                output = model(data)
                loss   = criterion(output, target)
                loss.backward()
                optimizer.step()
                epoch_losses.append(loss.item())

        # 查询当前已使用的隐私预算
        epsilon = privacy_engine.get_epsilon(target_delta)
        mean_loss = np.mean(epoch_losses)

        print(
            f"Epoch {epoch:3d}/{n_epochs} | "
            f"Loss: {mean_loss:.4f} | "
            f"Privacy: (ε={epsilon:.2f}, δ={target_delta})"
        )
        history.append({"epoch": epoch, "loss": mean_loss, "epsilon": epsilon})

    final_epsilon = privacy_engine.get_epsilon(target_delta)
    print(f"\nFinal privacy budget spent: ε={final_epsilon:.3f}, δ={target_delta}")
    return {"epsilon": final_epsilon, "delta": target_delta, "history": history}

实际部署考虑

DP-SGD 需要每样本梯度计算，这比标准的批处理梯度计算更昂贵 — 使用 Opacus 等库时通常内存开销 2–3 倍， 计算时间 1.5–2 倍。 某些层类型（BatchNorm）与 DP-SGD 不兼容，因为它们混合了 跨样本信息；必须替换为 GroupNorm 或 LayerNorm。 对于非常大的模型（LLM），DP 微调比从头开始 DP 预训练更实用： 使用 DP 微调预训练模型需要更少的梯度步骤， 因此隐私预算使用效率更高。

DP 提供形式化、量化的保证 — 而不仅仅是"我们添加了噪声，似乎更难 攻击。"当监管机构或法律机构问"这个模型有多隐私？"时， 一个具有 ε = 1.0, δ = 10-5 的训练模型能给出精确、可审计的答案。 这是相对于启发式防御的显著优势，也是 DP 越来越被隐私法规要求 用于敏感 ML 部署的原因。 [Opacus 教程 — Meta AI]

推荐阅读 关于 DP 在机器学习中的全面处理，请参见 Opacus 文档和 Abadi et al. (2016), "Deep Learning with Differential Privacy" (arXiv:1607.00133)。 关于 LLM 中隐私-效用权衡的专门讨论，请参见 Yu et al. (2022), "Differentially Private Fine-Tuning of Language Models" (arXiv:2110.06500)。

模块总结

本模块涵盖了模型提取和推理攻击的完整生命周期 — 从基于 API 的模型克隆的经济学，到影子模型的技术机制、 训练数据提取和针对加密流量的侧信道攻击， 再到速率限制、差分隐私、水印和异常检测的防御全景。

关键要点：

  • 模型提取在经济上是不对称的：攻击者可以用数千美元的 API 查询成本克隆价值数百万美元的训练成果。
  • 从 LLM 中提取训练数据并非理论问题 — Carlini et al. 在 GPT-2 和 ChatGPT 生产系统上进行了成功演示。
  • 成员推理将模型输出转化为隐私检测器，对 GDPR、HIPAA 以及医疗/金融 ML 具有实际影响。
  • 侧信道攻击（Whisper Leak、令牌长度、推测性解码、KV 缓存时序）表明仅靠 HTTPS 加密对于隐私敏感的 LLM 部署是不够的。
  • 硬件攻击（TPUXtract）表明物理接近加速器可以在没有任何 API 访问的情况下泄露架构秘密。
  • 差分隐私是唯一具有形式化、量化保证的防御 — 以效用和计算开销为代价。
  • 分层防御（速率限制 + 输出扰动 + 水印 + DP）比任何单一机制更鲁棒。
